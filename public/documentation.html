Movie_API Documentation

Movie_API is the server-side component of a web application named myFlix. The web application will provide users with access to information about different movies, directors and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Req. Number Request URL HTTP method Request body data format Response body data format
0 HOME / None None A confirmation message saying: Welcome to MyFlix!
1 Return a list of ALL movies to the user /movies GET None A JSON object holding data about all the movies
2 Return data about a single movie by title to the user /movies/:title GET None

A JSON object holding data about the movie to show, structured like:

{
Title: {type: String, required: true}
Description: {type: String, required: true}
Genre: {
Name: String
Description: String
}
Director: {
Name: String
Bio: String
}
Actors: [String]
ImagePath: String
Featured: Boolean (true/false)
}
3 Return a list of ALL genres to the user /genres GET None A JSON object holding data about all the genres
4 Return data about a genre by name /genres/:Name GET None

A JSON object holding data about the genre to show, structured like:

{
Name: {type: String, required: true}
Description: {type: String, required: true}
}
5 Return data about a director by name /directors/:Name GET None

A JSON object holding data about the director to show, structured like:

{
Name: {type: String, required: true}
Bio: {type: String, required: true}
Birthday: Date
Deathday: Date
}
6 Return data about a user by username /users/:Username GET None

A JSON object holding data about the user to show, structured like:

{
Username: {type: String, required: true}
Password: {type: String, required: true}
Email: {type: String, required: true}
Birthday: Date
FavoriteMovies: [{type: mongoose.Schema.Types.ObjectId, ref: "Movie"}]
}
7 Allow new users to register /users/ POST

A JSON object holding data about the user to add, structured like:

{
Username: {type: String, required: true}
Password: {type: String, required: true}
Email: {type: String, required: true}
Birthday: Date
FavoriteMovies: [{type: mongoose.Schema.Types.ObjectId, ref: "Movie"}]
}

A JSON object holding data about the user added, structured like:

{
Username: {type: String, required: true}
Password: {type: String, required: true}
Email: {type: String, required: true}
Birthday: Date
FavoriteMovies: [{type: mongoose.Schema.Types.ObjectId, ref: "Movie"}]
}
8 Allow users to add a movie to their list of favorites /users/:Username/movies/:MovieID POST None

A JSON object holding data about the user info with the movie added, structured like:

{
Username: {type: String, required: true}
Password: {type: String, required: true}
Email: {type: String, required: true}
Birthday: Date
FavoriteMovies: [{type: mongoose.Schema.Types.ObjectId, ref: "Movie"}]
}
9 Allow users to update their info by username /users/:Username PUT

A JSON object holding data about the user info to update, structured like:

FavoriteMovies: [{type: mongoose.Schema.Types.ObjectId, ref: "Movie"}]

10 Allow users to remove a movie from their list of favorites /users/:Username/movies/:MovieID DELETE None

A JSON object holding data about the user already without the removed movie, structured like:

{
Username: {type: String, required: true}
Password: {type: String, required: true}

Birthday: Date

11 Allow existing users to deregister /users/:Username DELETE None A confirmation message indicating that the user account has been deleted
